登录
====

## Use Case Name

登录

## Brief Description

这个用例描述用户如何登录

## Flow of Events

#### Basic Flow

这个用例开始于用户需要登录到网站享受服务时

1. 用户填写登录信息：手机号，密码
2. 系统进行表单验证
3. 表单验证成功后，用户成功登录

#### Alternative Flows

- 登录信息错误

	1. 提醒用户修改登录信息
	2. 重新登录

## Special Requirements

(无)

## Pre-Conditions

必须在当前电脑没有登录过用户的情况下才能进行登录

## Post-Conditions

登录成功后，返回登录前界面

## Extension Points

(无)
